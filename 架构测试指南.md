# 工业AI Agent 架构测试指南

## 📋 测试概述

本测试套件验证架构重构的6个关键要点：

1. ✅ **IntentRouter 只做归类，不做分支跳转**
2. ✅ **所有模块统一调度 - 都走 ModuleDispatcher**
3. ✅ **单一线性流水线 - 6步固定流程**
4. ✅ **全局统一基础设施 - 所有服务共享**
5. ✅ **没有硬编码 - 全部LLM动态生成**
6. ✅ **完整架构收敛 - 从三套体系收敛到一套**

## 🚀 快速开始

### 1. 编译项目

```bash
dotnet build --configuration Release
```

### 2. 运行测试

```bash
python architecture_test.py
```

## 📊 测试说明

### 测试1: IntentRouter 归类测试

**目标**: 验证IntentRouter只做归类不做分支跳转

**测试用例**:

- 简单对话: "你好", "我叫张三", "谢谢你"
- 工业诊断: "检查温度", "机床主轴异常", "诊断故障原因"

**验证点**:

- Intent正确归类为SimpleChat或IndustrialDiagnostic
- 没有硬编码的分支跳转逻辑

### 测试2: ModuleDispatcher 统一调度测试

**目标**: 验证所有模块都通过ModuleDispatcher调度

**测试模块**: 1-7 全部模块

**验证点**: 每个模块启动都显示"启动模块"标记

### 测试3: 线性流水线6步流程测试

**目标**: 验证固定6步流水线完整执行

**流水线步骤**:

1. \[1/6] 预处理完成
2. \[2/6] 意图归类完成
3. \[3/6] 上下文加载完成
4. \[4/6] 业务执行完成
5. \[5/6] 会话保存完成
6. \[6/6] 结果输出完成

**验证点**: 所有步骤标记都在输出中出现

### 测试4: 统一基础设施测试

**目标**: 验证SessionService、MemoryService、ToolService被共享使用

**测试点**:

- MemoryService: 记忆跨对话保持
- SessionService: 对话历史可查询
- ToolService: 工具被正常调用

### 测试5: 无硬编码测试

**目标**: 验证所有回复都由LLM动态生成

**检查点**:

- 没有发现旧代码中的硬编码回复
- 相同输入的两次回复不同(证明动态生成)

### 测试6: 架构收敛测试

**目标**: 验证从三套体系收敛到一套

**文件检查**:

- 删除旧文件: SimpleChatHandler、IndustrialDiagnosticHandler等
- 存在新文件: UnifiedDialogModule、AgentDialog
- ModuleType从9个模块收敛到7个

## 🔧 故障排除

### 问题1: 找不到编译文件

```
解决: 运行 dotnet build --configuration Release
```

### 问题2: LLM响应超时

```
解决: 确保Ollama服务正常运行，检查ModelConfig配置
```

### 问题3: 测试不稳定

```
解决: 增加测试等待时间，检查网络/LLM服务稳定性
```
