# Agent1 可追溯性架构优化方案

## 📋 概述

本次优化为Agent1项目增加了完整的可追溯性（Tracing）功能，实现了调用链路的可视化追踪，同时保持了原有架构的完整性和非侵入性。

## 🎯 实现目标

### ✅ 已完成功能

1. **唯一TraceID生成** - 每次用户问答生成唯一标识，贯穿全流程
2. **精确时间记录** - 记录每个模块/服务的执行耗时
3. **输入输出追踪** - 完整记录每个环节的输入输出数据
4. **文件/数据访问记录** - 记录访问的文件路径和数据片段
5. **树状调用链路可视化** - 通过Web界面直观展示完整调用链路
6. **按TraceID查询** - 支持通过唯一标识查询历史追踪记录
7. **可配置开关** - 支持通过配置启用/禁用追踪功能
8. **内存+本地文件存储** - 无需外部数据库，便于调试

## 🏗️ 架构设计

### 目录结构

```

│   ├── Models/

│   │   ├── TraceStatus.cs           # 追踪状态枚举
│   │   ├── Span.cs                  # 单个调用跨度
│   │   └── Trace.cs                 # 完整追踪记录
│   ├── Web/
│   │   └── TracingWebServer.cs      # 内置Web服务器
│   ├── TracingConfig.cs             # 追踪配置
│   ├── ITraceManager.cs             # 追踪管理器接口
│   ├── TraceManager.cs              # 追踪管理器实现
│   ├── TraceContext.cs              # 异步上下文

```

### 核心组件说明

#### 1. 数据模型层 (Tracing/Models/)

**Trace.cs**
- 表示一次完整的用户交互追踪
- 包含：TraceID、开始/结束时间、状态、用户输入、最终输出、根Span
- 支持树状结构查找任意Span

**Span.cs**
- 表示单个调用环节
- 包含：SpanID、父SpanID、名称、类型、时间、输入输出、错误信息
- 支持记录文件路径和数据片段
- 支持嵌套子Span形成树状结构

**SpanType.cs**
- UserInput: 用户输入
- Scheduler: 调度器
- Module: 范式模块
- Service: 服务
- FileAccess: 文件访问
- DataAccess: 数据访问
- LlmCall: LLM调用
- ToolCall: 工具调用
- Output: 最终输出

#### 2. 管理层 (Tracing/)

**TraceManager**
- 核心管理类，实现ITraceManager接口
- 负责Trace的创建、查询、更新
- 管理Span的生命周期
- 处理内存存储和文件持久化
- 线程安全设计（ConcurrentDictionary）

**TraceContext**
- 使用AsyncLocal存储当前上下文
- 支持异步环境下的TraceID和SpanID传递
- 无需手动传递参数

**TracedModuleDispatcher**
- 装饰器模式实现
- 包装原有的ModuleDispatcher
- 自动记录模块执行的开始和结束
- 非侵入式设计

#### 3. Web可视化层 (Tracing/Web/)

**TracingWebServer**
- 使用.NET原生HttpListener
- 无需ASP.NET Core框架依赖
- 提供：
  - `/` - 可视化Dashboard
  - `/api/traces` - 追踪列表API
  - `/api/traces/{traceId}` - 详情API

### 集成方式

1. **装饰器模式** - 通过TracedModuleDispatcher包装原有调度器
2. **配置驱动** - 通过TracingConfig控制所有行为
3. **资源管理** - 使用IDisposable正确清理Web服务器和资源

## 🔧 配置说明

### TracingConfig 配置项

```csharp
var config = new TracingConfig
{
    Enabled = true,                    // 是否启用追踪
    PersistToFile = true,              // 是否持久化到文件
    StorageDirectory = "traces",       // 文件存储目录
    MaxTracesInMemory = 100,           // 内存最大保留数量
    WebPort = 5000,                    // Web UI端口
    EnableWebUI = true                 // 是否启用Web UI
};
```

## 🔒 架构保护原则

### ✅ 遵守的原则

1. **不破坏现有分层架构**
   - 表现层 → 调度层 → 应用服务层 → 基础设施层保持完整
   - Tracing作为独立模块平行存在

2. **不修改四大范式核心逻辑**
   - CoT/ReAct/Reflection/RAG模块完全未改动
   - 通过装饰器模式实现追踪

3. **不影响接口抽象**
   - IInferenceModule、IModuleFactory等接口保持原样
   - 工厂模式、调度器设计未改变

4. **独立目录**
   - 所有新增代码在Tracing/目录下
   - 与现有Services/Modules完全隔离

5. **依赖注入扩展**
   - 通过装饰器而非硬编码
   - 支持随时移除而不影响功能

## 📈 性能影响

1. **可配置开关** - 生产环境可完全禁用，零性能损耗
2. **异步设计** - Web服务器在后台线程运行
3. **内存限制** - 通过MaxTracesInMemory控制内存占用
4. **文件异步** - 持久化操作使用安全的写入方式

## 🔄 扩展指南

### 添加自定义Span追踪

```csharp
// 1. 获取当前TraceID
var traceId = TraceContext.CurrentTraceId;

// 2. 创建Span
var span = traceManager.StartSpan(
    traceId,
    "MyOperation",
    SpanType.Service,
    TraceContext.CurrentSpanId,
    "input data"
);

try
{
    // 3. 执行业务逻辑
    var result = DoSomething();
    
    // 4. 完成Span
    traceManager.CompleteSpan(traceId, span.SpanId, result);
}
catch (Exception ex)
{
    // 5. 标记失败
    traceManager.FailSpan(traceId, span.SpanId, ex.Message);
    throw;
}
```

### 添加自定义元数据

```csharp
traceManager.UpdateSpanMetadata(traceId, spanId, "key", "value");
```

### 记录文件访问

```csharp
traceManager.SetSpanFilePath(traceId, spanId, @"C:\data\file.txt");
```
