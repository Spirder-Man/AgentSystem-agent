# 多轮对话系统架构设计

## 一、系统概述

本系统实现了一个智能多轮对话 Agent，支持记忆机制、工具自动调用、多范式整合。

---

## 二、记忆机制设计

### 2.1 核心概念

#### 记忆的本质：存储和检索对话历史的能力。

### 2.2 架构设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                        SessionManager                             │
│  ┌───────────────────────────────────────────────────────────┐   │
│  │  SessionContext                                          │   │
│  │  - SessionId: 唯一会话标识                      │   │
│  │  - DialogHistory: 对话历史列表               │   │
│  │  - SessionType: 会话类型(通用/工业)          │   │
│  │  - UserPromptTemplate: 用户自定义Prompt         │   │
│  └───────────────────────────────────────────────────────────┘   │
│  ┌───────────────────────────────────────────────────────────┐   │
│  │  DialogTurn (单个对话轮次                             │   │
│  │  - Role: 用户/助手                                │   │
│  │  - Content: 内容                                  │   │
│  │  - Timestamp: 时间戳                             │   │
│  │  - ToolCalls: 工具调用记录                      │   │
│  └───────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 记忆管理机制实现要点

| 功能 | 说明 |
|------|------|
| **会话创建 | 独立存储 | `CreateSession() | 创建新会话 |
| **历史追加 | `AddDialogTurn()` | 添加新的对话轮次 |
| **历史检索 | `GetFormattedHistory()` | 获取格式化历史 |
| **摘要检索 | `GetContextSummary()` | 获取简短上下文摘要 |
| **清空历史 | `ClearHistory()` | 清空当前会话历史 |

### 2.4 Prompt注入方式

历史以格式化历史通过 Prompt 注入：

```
【对话历史记录】
👤 用户 [10:30:15]
你好，你叫什么名字？
────────────────────────────────────────
🤖 助手 [10:30:18]
你好！我是工业AI助手...
────────────────────────────────────────
```

## 三、工具调用通用范式设计

### 3.1 三段式范式流程

```
┌─────────────────────────────────────────────────────────────┐
│  阶段1：分析与规划                                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 输入: 历史 + 用户问题 + 工具列表       │    │
│  │ 输出: 工具规划(需要/不需要 + 工具列表 │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段2：执行工具调用                                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 逐个执行工具列表                              │    │
│  │ 错误处理 + 超时保护                         │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  阶段3：生成最终回复                                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 输入: 历史 + 问题 + 工具结果              │    │
│  │ 输出: 最终用户回复                        │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 工具规划解析

**结构化输出：

```
思考: [分析内容]
需要工具: [是/否]
工具列表: [工具1, 工具2]
```

### 3.3 工具调用实现

```csharp
// 工具调用处理策略
1. 简单工具: GetSpindleTemperature()
2. 参数工具: Calculate(1+1), WebSearch(查询词)
```

## 五、关键技术实现

### 5.1 记忆检索优化

| 方法 | 说明 |
|------|------|
| **最近优先** | 只取最近N轮，避免上下文过长
| **格式化** | 👤 用户 🤖 助手标签，提高可读性
| **角色区分** | 区分用户和助手的不同角色
| **时间戳** | 精确到秒的时间记录

### 5.2 工具调用安全机制

1. **工具验证** - 验证工具名是否有效
2. **参数提取** - 从工具名里提取参数
3. **异常处理** - 工具失败时的错误处理
4. **结果保存** - 保存工具调用结果

### 5.3 用户命令系统

| 命令 | 功能 |
|------|------|
| `exit` / `退出` | 退出当前对话 |
| `clear` / `清空` | 清空对话历史 |
| `history` / `历史` | 查看对话历史 |

## 六、使用场景示例

### 场景1：工业设备诊断

```
用户: 机床主轴温度异常
↓
阶段1: 分析需要调用工具
阶段2: 调用 GetSpindleTemperature, GetTemperatureThreshold
阶段3: 基于温度和阈值分析，输出结论
```

### 场景2：普通闲聊

```
用户: 你好，介绍下你是谁
↓
阶段1: 分析不需要工具
阶段2: 跳过
阶段3: 基于历史和知识直接回复
```

### 场景3：复杂多轮对话

```
用户1: 告诉我当前时间
↓
调用 GetCurrentTime → 返回时间
↓
用户2: 那现在温度呢
↓
基于历史: 用户问了时间
↓
调用 GetSpindleTemperature
↓
回复综合信息
```
