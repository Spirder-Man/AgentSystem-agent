# 架构整改方案

## 一、整改背景

当前项目存在硬编码问题，需要进行架构整改以适配化工园区危化品合规审核场景。

## 二、整改内容

### 2.1 配置模型整改

- 将硬编码配置迁移到AppConfig
- 增加化工场景专用配置类
- 支持多环境配置切换

### 2.2 服务层整改

- LlmService重试参数配置化
- 工具列表支持外部配置
- 知识库服务扩展

### 2.3 新增服务

- IntegrationService - 工业系统集成
- AuditService - 操作审计

### 2.4 新增模块

- ComplianceCheckModule - 合规审查

## 三、整改优先级

| 优先级 | 整改项 | 工作量 |
|--------|--------|--------|
| P0 | 配置模型整改 | 中 |
| P1 | 服务层整改 | 大 |
| P2 | 新增IntegrationService | 中 |
| P2 | 新增AuditService | 中 |
| P3 | 新增ComplianceCheckModule | 大 |

## 四、风险评估

- 整改过程中需保证现有功能不受影响
- 需要充分的回归测试
- 需要更新相关文档
