# 多轮对话系统架构设计

## 一、系统概述

本系统实现了一个智能多轮对话 Agent，支持记忆机制、工具自动调用、多范式整合。

## 二、记忆机制设计

### 2.1 核心概念

**记忆的本质：存储和检索对话历史的能力。**

### 2.2 架构设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                        SessionManager                             │
│  ┌───────────────────────────────────────────────────────────────┐   │
│  │  SessionContext                                          │   │
│  │  - SessionId: 唯一会话标识                      │   │
│  │  - DialogHistory: 对话历史列表               │   │
│  │  - SessionType: 会话类型(通用/工业)          │   │
│  │  - UserPromptTemplate: 用户自定义Prompt         │   │
│  └───────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 记忆管理机制

| 功能 | 说明 |
|------|------|
| 会话创建 | `CreateSession()` 创建新会话 |
| 历史追加 | `AddDialogTurn()` 添加新的对话轮次 |
| 历史检索 | `GetFormattedHistory()` 获取格式化历史 |
| 摘要检索 | `GetContextSummary()` 获取简短上下文摘要 |
| 清空历史 | `ClearHistory()` 清空当前会话历史 |

## 三、工具调用通用范式设计

### 3.1 三段式范式流程

```
阶段1：分析与规划 → 阶段2：执行工具调用 → 阶段3：生成最终回复
```

### 3.2 工具规划解析

结构化输出：
```
思考: [分析内容]
需要工具: [是/否]
工具列表: [工具1, 工具2]
```

## 五、核心服务设计

### 5.1 LlmService

负责与LLM交互，支持流式输出和同步输出。

### 5.2 SessionService

管理会话历史，支持多轮对话上下文保持。

### 5.3 KnowledgeBaseService

提供BM25检索，支持化工知识库查询。

### 5.4 ToolService

管理工业工具，支持工具自动调用。
