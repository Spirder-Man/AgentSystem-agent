# 架构优化方案

## 可追溯性架构优化

本次优化为Agent1项目增加了完整的可追溯性（Tracing）功能，实现了调用链路的可视化追踪。

## 实现目标

1. **唯一TraceID生成** - 每次用户问答生成唯一标识，贯穿全流程
2. **精确时间记录** - 记录每个模块/服务的执行耗时
3. **输入输出追踪** - 完整记录每个环节的输入输出数据
4. **文件/数据访问记录** - 记录访问的文件路径和数据片段
5. **树状调用链路可视化** - 通过Web界面直观展示完整调用链路
6. **按TraceID查询** - 支持通过唯一标识查询历史追踪记录
7. **可配置开关** - 支持通过配置启用/禁用追踪功能
8. **内存+本地文件存储** - 无需外部数据库，便于调试

## 核心组件

### Tracing/Models/

- **Trace.cs** - 表示一次完整的用户交互追踪
- **Span.cs** - 表示单个调用环节
- **SpanType.cs** - 跨度类型枚举

### Tracing/

- **TraceManager** - 核心管理类，负责Trace的创建、查询、更新
- **TraceContext** - 使用AsyncLocal存储当前上下文
- **TracedModuleDispatcher** - 装饰器模式的调度器
- **TracingWebServer** - 内置Web可视化服务器

### _spanType枚举值

- UserInput: 用户输入
- Scheduler: 调度器
- Module: 范式模块
- Service: 服务
- FileAccess: 文件访问
- DataAccess: 数据访问
- LlmCall: LLM调用
- ToolCall: 工具调用
- Output: 最终输出
