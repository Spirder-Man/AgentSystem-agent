| 层级 | 组件 | 状态 |

| **服务层** | LLM、知识库、数据库、审计、会话、集成服务 | 已实现 |
| **配置层** | 统一配置中心 AppConfig（化工行业专用） | 已实现 |
| **数据层** | PostgreSQL + pgvector 混合检索 | 已实现 |

## 三、下一步计划（建议优先级排序）

### Phase 1: 基础设施加固（紧急，1-2周）

#### 1. 引入依赖注入（DI）
- 使用 `Microsoft.Extensions.DependencyInjection`
- 将所有 Service 注册到 DI 容器
- Program.cs 改为 Host Builder 模式
- **收益**: 解耦服务依赖，提升可测试性

#### 2. 配置外部化
- 创建 `appsettings.json` + `appsettings.Development.json`
- 敏感信息迁移到环境变量或 User Secrets
- 使用 `IConfiguration` 绑定配置类
- **收益**: 消除安全隐患，支持多环境部署

#### 3. 引入结构化日志
- 集成 Serilog + Console/File Sink
- 替换所有 `Console.WriteLine` 调试输出
- 添加请求追踪（CorrelationId）
- **收益**: 生产环境可观测性

### Phase 2: 核心功能完善（重要，2-3周）

#### 4. 审计日志持久化
- 将 AuditService 日志写入 PostgreSQL `audit_logs` 表
- 实现 180 天自动清理策略
- **收益**: 满足等保三级合规要求

#### 5. 向量检索完整链路
- 完善文档嵌入生成流程（调用 nomic-embed-text）
- 实现知识库增量更新机制（基于文件哈希去重）
- 优化 HNSW 索引参数（基于实际数据量调优）
- **收益**: RAG 检索质量提升

#### 6. 扩展测试覆盖
- 为 ComplianceCheckModule 添加集成测试
- 为 HybridKnowledgeBaseService 添加混合检索测试
- 为 ModuleDispatcher 添加端到端测试
- 目标: 核心路径测试覆盖率 > 70%
- **收益**: 回归保护，重构信心

### Phase 3: 对外能力建设（中期，3-4周）

#### 7. 暴露 Web API
- 使用 ASP.NET Core Minimal API 或 Controller
- 提供 `/api/chat`（对话）、`/api/compliance/check`（合规检查）等端点
- 添加 Swagger/OpenAPI 文档
- **收益**: 支持前端对接和第三方集成

#### 8. 会话持久化
- 将 SessionService 从内存迁移到 PostgreSQL
- 支持跨重启的会话恢复
- **收益**: 生产可用性

### Phase 4: 高级特性（远期）

#### 9. Milvus 向量数据库接入
- 替代或补充 pgvector，提升大规模检索性能
- **前置条件**: Phase 2 完成

#### 10. ERP/WMS/EHS 系统对接
- 实现 IntegrationService 的实际连接逻辑
- **前置条件**: Web API 层就绪

#### 11. 多模型支持
- 支持在线模型（如 Claude API、通义千问）作为备选推理引擎
- 模型路由策略（本地模型 → 在线模型降级）
