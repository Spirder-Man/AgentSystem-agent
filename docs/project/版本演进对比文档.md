## V0.1 - 第一版：四大基础范式

### 设计目标

建立一套通用的工业AI对话流程框架，确保输出的严谨性和可控性。

### 四大基础范式

| 范式 | 英文 | 说明 | 特点 |
|------|------|------|------|
| 1️⃣ | **ReAct** | 推理 + 行动 + 观察循环 | 透明的工具调用过程，工具结果可验证 |
| 2️⃣ | **ToC** | 内容大纲 + 结构化输出 | 结构清晰，符合工业报告习惯 |
| 3️⃣ | **反思纠错** | 自我检查 + 修正输出 | 输出质量高，减少幻觉 |
| 4️⃣ | **CoT (思维链)** | 逐步思考过程 | 过程透明，可调试 |

### 完整工作流程（6阶段）

```
用户输入
    ↓
阶段1: 分析与规划工具（ReAct）
    ↓ 输出思考过程（CoT）
阶段2: 执行工具调用（ReAct）
    ↓ 逐个调用工具并显示结果
阶段3: 生成内容大纲（ToC）
    ↓ 确定输出结构
阶段4: 生成初步结论
    ↓ 按ToC结构填充内容
阶段5: 自我反思与纠错（反思）
    ↓ 检查数据真实性、结论严谨性等
阶段6: 生成最终结论
    ↓ 基于反思修正问题
用户输出（含所有中间过程）
```

### 存在的问题

| 问题 | 现象 | 影响 |
|------|------|------|
| 1️⃣ 记忆不工作 | 问"刚才的温度"还重新调用工具 | 用户体验差，浪费资源 |
| 2️⃣ 输出太冗长 | 6阶段全透明给用户看，每次等很久 | 响应慢，用户体验差 |
| 3️⃣ 格式混乱 | 还有残留 `<think>` 标签，内容重复 | 输出不整洁 |
| 4️⃣ 工具滥用 | 用户只是自我介绍，也要调用3个工具 | 浪费资源，响应慢 |

#### 3. `<think>`标签过滤

**问题：** DeepSeek模型内部思考用`<think>`标签包裹，但偶尔会显示给用户

**解决方案：**
```csharp
// 流式输出时实时过滤
while (true)
{
    int thinkStart = buffer.IndexOf("<think>");
    int thinkEnd = buffer.IndexOf("</think>");
    
    // 遇到<think>开始标签，跳过
    if (thinkStart >= 0 && (thinkEnd < 0 || thinkStart < thinkEnd))
    {
        isInThinkBlock = true;
        buffer = buffer.Substring(thinkStart + "<think>".Length);
    }
    // 遇到</think>结束标签，恢复输出
    else if (isInThinkBlock && thinkEnd >= 0)
    {
        buffer = buffer.Substring(thinkEnd + "</think>".Length);
        isInThinkBlock = false;
    }
    else
    {
        break;
    }
}

// 最后清理残留
var finalResult = result.ToString()
    .Replace("<think>", "")
    .Replace("</think>", "")
    .Trim();
```

#### 2. 重试机制（Retry Logic）

**问题：** 本地模型调用偶尔超时或失败

**解决方案：**
```csharp
private const int MaxRetries = 3;
private const int RetryDelayMs = 1000;

private async Task<string> InvokeStreamWithRetryAsync(string prompt, ConsoleColor color, string stageName = "")
{
    Exception lastException = null;
    
    for (int attempt = 1; attempt <= MaxRetries; attempt++)
    {
        try
        {
            if (attempt > 1)
            {
                Console.WriteLine($"\n🔄 第{attempt}次重试 {stageName}...");
            }
            
            return await InvokeStreamAsync(prompt, color);
        }
        catch (OperationCanceledException ex) // 超时
        {
            lastException = ex;
            Console.WriteLine($"\n⏰ 请求超时 ({attempt}/{MaxRetries}): {ex.Message}");
            
            if (attempt < MaxRetries)
                await Task.Delay(RetryDelayMs);
        }
        catch (Exception ex) // 其他错误
        {
            lastException = ex;
            Console.WriteLine($"\n❌ 错误 ({attempt}/{MaxRetries}): {ex.Message}");
            
            if (attempt < MaxRetries)
                await Task.Delay(RetryDelayMs);
        }
    }
    
    Console.ForegroundColor = ConsoleColor.Red;
    Console.WriteLine($"\n❌ 所有重试失败: {lastException?.Message}");
    Console.ResetColor();
    
    return $"生成失败: {lastException?.Message}";
}
```

#### 3. 工具调用容错匹配增强

**问题：** 模型输出工具名称格式不统一（有时带括号，有时小写，有时拼错）

**解决方案：**
```csharp
private async Task<string> CallToolAsync(string toolName)
{
    toolName = toolName.Trim();
    var lowerTool = toolName.ToLower();
    
    // 容错匹配1：关键词匹配
    if (lowerTool.Contains("spindle") && lowerTool.Contains("temperature"))
        return _tools.GetSpindleTemperature();
    if (lowerTool.Contains("threshold"))
        return _tools.GetTemperatureThreshold();
    if (lowerTool.Contains("time") && lowerTool.Contains("current"))
        return _tools.GetCurrentTime();
    
    // 容错匹配2：带参数的工具
    if (lowerTool.Contains("calculate") || lowerTool.Contains("计算"))
    {
        var idx = toolName.IndexOf('(');
        if (idx > 0)
        {
            var arg = toolName.Substring(idx + 1).TrimEnd(')');
            return _tools.Calculate(arg);
        }
    }
    
    // 容错匹配3：精确匹配（包括各种写法）
    return toolName switch
    {
        "GetSpindleTemperature" => _tools.GetSpindleTemperature(),
        "GetTemperatureThreshold" => _tools.GetTemperatureThreshold(),
        "GetCurrentTime" => _tools.GetCurrentTime(),
        "GetSpindleTemperature()" => _tools.GetSpindleTemperature(),
        "GetTemperatureThreshold()" => _tools.GetTemperatureThreshold(),
        "GetCurrentTime()" => _tools.GetCurrentTime(),
        "CurentTime" => _tools.GetCurrentTime(), // 拼写错误也能识别
        _ => $"未知工具: {toolName}"
    };
}
```

#### 4. 工具规划解析容错增强

**问题：** 模型输出格式不统一（有时写"需要工具: 是"，有时直接写工具列表）

**解决方案：**
```csharp
private ToolPlan ParseToolPlan(string analysis)
{
    var plan = new ToolPlan();
    
    try
    {
        // 1. 增加调试输出，可看到每一步解析
        Console.WriteLine("\n🔍 解析调试:");
        Console.WriteLine("分析内容长度: " + analysis.Length);
        
        var lines = analysis.Split('\n');
        foreach (var line in lines)
        {
            var trimmed = line.Trim();
            
            if (!string.IsNullOrWhiteSpace(trimmed))
                Console.WriteLine($"  行: {trimmed}");
            
            // 2. 解析"需要工具:"
            if (trimmed.StartsWith("需要工具:"))
            {
                var value = trimmed.Substring("需要工具:".Length).Trim();
                
                // 支持"是"字，也支持直接是工具列表
                plan.NeedsTools = value.Contains("是") || !string.IsNullOrWhiteSpace(value);
                
                // 如果"需要工具:"后面直接是工具列表，也解析
                if (!value.Contains("是") && !value.Contains("否") && !string.IsNullOrWhiteSpace(value))
                {
                    var tools = value.Split(',')
                                    .Select(t => t.Trim())
                                    .Where(t => !string.IsNullOrWhiteSpace(t) && !t.Contains("无"))
                                    .ToList();
                    if (tools.Count > 0)
                        plan.ToolNames.AddRange(tools);
                }
            }
            // 3. 解析"工具列表:"
            else if (trimmed.StartsWith("工具列表:"))
            {
                var value = trimmed.Substring("工具列表:".Length).Trim();
                if (!string.IsNullOrWhiteSpace(value))
                {
                    var tools = value.Split(',')
                                    .Select(t => t.Trim())
                                    .Where(t => !string.IsNullOrWhiteSpace(t) && !t.Contains("无"))
                                    .ToList();
                    if (tools.Count > 0)
                        plan.ToolNames.AddRange(tools);
                }
            }
        }
        
        // 4. 只要工具列表有内容，强制设置NeedsTools=true
        if (plan.ToolNames.Count > 0)
            plan.NeedsTools = true;
        
        // 5. 去重
        plan.ToolNames = plan.ToolNames.Distinct().ToList();
    }
    catch (Exception ex)
    {
        Console.WriteLine($"❌ 解析错误: {ex.Message}");
        plan.NeedsTools = false;
        plan.ToolNames = new List<string>();
    }
    
    Console.WriteLine($"  结果: NeedsTools={plan.NeedsTools}, Tools=[{string.Join(",", plan.ToolNames)}]");
    return plan;
}
```

### 第二层：问题复杂度判断（IsSimpleQuestion）

```csharp
private bool IsSimpleQuestion(string input)
{
    string lower = input.ToLower();
    
    // 第一类：问候语
    string[] greetingPatterns = { "你好", "hi", "hello", "在吗", "忙吗", ... };
    
    // 第二类：自我介绍
    string[] introPatterns = { "我叫", "我是", "名字是" };
    
    // 第三类：简单追问
    string[] followUpPatterns = { "哪个", "什么", "为什么", "刚才", "之前" };
    
    // 判断逻辑
    bool isGreeting = greetingPatterns.Any(p => lower.Contains(p));
    bool isIntro = introPatterns.Any(p => lower.Contains(p));
    
    if (isGreeting || isIntro)
        return true; // → 简单模式
    
    if (input.Length < 15 && followUpPatterns.Any(p => lower.Contains(p)))
        return true; // → 简单模式
    
    return false; // → 完整模式
}
```

## 常见问题解答

### Q1: 工业对话会走四大范式吗？

**A: 不会！**

四大范式（CoT/ReAct/ToC/反思）是V0.1版本的功能，对应菜单选项1-6。

V1.0/V2.0的SmartDialogSystem（菜单选项7-8）是独立设计的另一套系统，**不走四大范式**。

| 菜单选项 | 功能 | 说明 |
|---------|------|------|
| 1-6 | 四大范式 | `CoT.cs`, `RAG.cs` 等 |
| 7-8 | SmartDialogSystem | `SmartDialogSystem.cs` |

---

### Q2: 工具调用判断太模糊？

**A: 确实，但这是有意为之！**

工具调用判断不是硬编码，而是：
1. 把问题、历史、工具列表发给LLM
2. 让LLM自己判断是否需要工具
3. 解析LLM的结构化输出

**优点：** 灵活、可扩展
**缺点：** 解析可能失败（所以V2.0增加了大量容错逻辑）

---

### Q3: 记忆系统有几套？

**A: 两套！**

```
第一套：SessionManager（通用历史）
├─ DialogHistory - 完整对话历史（所有内容）
└─ GetFormattedHistory() - 格式化后发给LLM

第二套：SmartDialogSystem（关键事实）
├─ _userProfile - 用户档案（用户名、职位）
├─ _keyFacts - 关键事实库（温度、阈值）
└─ TryAnswerFromMemory() - 快速检索
```

---

### Q4: 为什么感觉混乱？

**A: 因为这是逐步迭代出来的！**

每个版本都保留了前一个版本的功能（菜单1-6还在），同时又加了新功能（菜单7-8），所以看起来有重复。

**建议使用方式：**
- 日常对话 → 选7（通用对话）
- 工业诊断 → 选8（工业诊断）
- 体验范式 → 选1-6（学习用）

---

### Q5: 重试机制是重试什么？

**A: 只重试LLM调用，不重试工具调用！**

```
重试：InvokeStreamAsync()（调用LLM生成内容）
    ├─ 超时 → 重试
    ├─ 网络错误 → 重试
    └─ 其他异常 → 重试

不重试：ExecuteToolsAsync()（调用工具）
    └─ 工具调用失败就失败了，返回错误信息
```

---

### Q6: SmartDialogSystem会调用RAG吗？

**A: 不会！**

SmartDialogSystem是独立的，RAG是菜单选项6的功能，两者互不调用。
