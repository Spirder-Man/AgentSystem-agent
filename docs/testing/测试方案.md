# 测试方案

## 测试目标

1. 验证模块9（SmartAutoRouter）和模块8（SmartDialogIndustrial）的功能
2. 检测这两个模块是否正确调用四大范式
3. 测试简单对话模式和专业诊断模式的切换逻辑
4. 验证记忆机制是否正常工作

## 架构分析

### 模块定义 (ModuleType)

| 模块 | 名称 | 说明 |
|------|------|------|
| 1 | CoT (Solid Output) | 思维链推理 |
| 2 | CoT (Stream Output) | 思维链流式输出 |
| 3 | ReAct (Solid Output) | 推理-行动循环 |
| 4 | ReAct (Stream Output) | 推理-行动流式输出 |
| 5 | Reflection | 反思纠错 |
| 6 | RAG | 检索增强生成 |
| 7 | Smart Dialog (General) | 通用智能对话 |
| 8 | Smart Dialog (Industrial) | 工业智能对话 |
| 9 | Smart Auto Router | 自动语义路由 |

### 关键发现

- 模块9（SmartAutoRouter）和模块8（SmartDialogIndustrial）是独立系统
- 它们使用自己的Handler，不经过CoT/ReAct/Reflection/RAG
- 工业诊断使用ToolService调用工具，不是通过四大范式

## 测试场景

### 场景1：模块9 - 简单对话模式

**输入序列**：
1. "你好，我叫李工" → 提取名字
2. "我是谁？" → 从记忆回答
3. "你叫什么？" → 从记忆回答

**预期结果**：
- ⚡ 识别为：简单对话模式
- 记忆正确保存和回答
- 无特殊标签输出

### 场景2：模块9 - 专业诊断模式

**工业关键词**：`温度`、`机床`、`主轴`、`异常`、`故障`、`阈值`、`诊断`

**输入序列**：
1. "检查机床主轴温度" → 触发工业诊断
2. "温度超过阈值了吗？" → 继续工业诊断

**预期结果**：
- 📋 识别为：专业诊断模式
- 调用工业工具获取数据
- 生成诊断建议
