## 案例1：日常闲聊场景（通用对话模式）

### 场景说明
用户第一次用这个系统，先问候，自我介绍，然后闲聊几句，最后问点时间相关的问题。

### 对话步骤

| 轮次 | 👤用户输入 | ✅理想输出 | 预期验证点 |
|------|---------|---------|---------|
| 1 | 你好 | ```🤖：你好！我是SpirderMan，很高兴为您服务！有什么我可以帮忙的吗？```<br><br>✅ 验证点：<br>- 走 ⚡快速对话模式<br>- 不要调用工具<br>- 友好自然 | 快速模式、无工具调用 |
| 2 | 我叫李明，是一名工程师 | ```🤖：你好李明！很高兴认识你！工程师很厉害哦～有什么我可以帮忙的吗？```<br><br>✅ 验证点：<br>- 显示 📝记住了！用户名字: 李明<br>- 走快速对话模式<br>- 记住了用户身份 | 记忆功能（用户档案） |
| 3 | 我叫什么？ | ```🤖：你叫李明！是一名工程师～```<br><br>✅ 验证点：<br>- 🧠从记忆中找到答案！<br>- 不要调用工具<br>- 直接回答，速度快 | 记忆检索功能 |
| 4 | 现在几点了？ | ```（先走到完整诊断模式）```<br><br>【分析中】→ 思考后说需要工具<br><br>【调用工具】→ 🔧调用GetCurrentTime成功<br><br>【生成答案】→ 给出答案<br><br>✅ 验证点：<br>- 调用GetCurrentTime<br>- 显示当前时间<br>- 同时记住时间？ | 工具调用功能 |
| 5 | 今天天气怎么样？（可选，如果有WebSearch） | ```（如果WebSearch有API）```<br><br>调用WebSearch查询天气<br><br>✅ 验证点：<br>- 工具调用<br>- 天气查询 | WebSearch工具（可选） |
| 6 | 好的，谢谢 | ```🤖：不客气！有问题随时问我～```<br><br>✅ 验证点：<br>- 走快速对话模式<br>- 自然友好的结尾 | 结束友好自然 |

## 案例2：工业诊断场景（工业诊断模式）

### 场景说明
用户在工厂里，需要检查机床温度，发现异常，询问解决方案，最后确认是否记住。

### 对话步骤

| 轮次 | 👤用户输入 | ✅理想输出 | 预期验证点 |
|------|---------|---------|---------|
| 1 | 检查一下机床状态 | ```📋 完整诊断模式```<br><br>【分析中】→ 认为需要检查<br><br>【调用工具】→<br>🔧GetSpindleTemperature: 198℃<br>🔧GetTemperatureThreshold: ≤180℃<br><br>【生成答案】→<br>```🤖：您好！检查到主轴温度异常！当前温度: 198℃，安全阈值: ≤180℃，建议立即检查冷却系统。```<br><br>✅ 验证点：<br>- 📝记住了！主轴温度: 198℃<br>- 调用2个工具<br>- 给出诊断结论<br>- 友好建议 | 工具调用、自动记忆、诊断输出 |
| 2 | 刚才温度是多少？ | ```🧠从记忆中找到答案！```<br><br>```🤖：刚才检查到的温度是198℃！安全阈值是≤180℃，温度有点偏高哦～```<br><br>✅ 验证点：<br>- 直接从记忆回答<br>- 不调用工具<br>- 温度准确<br>- 同时提醒阈值 | 记忆检索（温度+阈值） |
| 3 | 现在温度还是一样吗？ | ```📋 完整诊断模式```<br><br>【调用工具】→ GetSpindleTemperature<br><br>✅ 验证点：<br>- 重新调用温度工具<br>- 给出最新温度<br>- 对比历史（如果需要） | 工具调用更新 |
| 4 | 应该怎么办？ | ```（可以从反思模式给出建议）```<br><br>```🤖：根据目前情况，建议您：1. 检查冷却液是否充足；2. 检查轴承是否磨损；3. 开启备用冷却系统；4. 持续监控温度变化。```<br><br>✅ 验证点：<br>- 给出具体可执行建议<br>- 建议符合工业场景 | 建议生成、工具数据支撑 |
| 5 | 好的，我去看看 | ```🤖：好的，请注意安全！如果有其他问题随时告诉我～```<br><br>✅ 验证点：<br>- 走快速模式<br>- 友好提醒安全 | 结束友好自然 |

## 快速测试小清单

运行后请检查以下项目：

### 基础功能
- [ ] 程序可以正常启动
- [ ] 可以选择选项7或8
- [ ] 对话可以正常输入和输出
- [ ] 对话历史计数正确（📊当前对话历史: x轮）

### 记忆功能
- [ ] 📝记住了！用户名字: xxx （案例1-2）
- [ ] 📝记住了！主轴温度: xxx （案例2-1）
- [ ] 🧠从记忆中找到答案！ （案例1-3、案例2-2）
- [ ] 记忆回答不调用工具 ✅不需要调用工具

### 工具调用
- [ ] 🔧调用GetSpindleTemperature成功
- [ ] 🔧调用GetCurrentTime成功
- [ ] 工具结果正确显示
- [ ] 只在需要时才调用工具（不是所有问题都调）

### 输出格式
- [ ] ⚡快速对话模式 （简单问题）
- [ ] 📋完整诊断模式 （复杂问题）
- [ ] 没有残留 `<think>` 标签
- [ ] 输出自然友好，不啰嗦

### 性能
- [ ] 记忆检索很快（<1秒）
- [ ] 工具调用不超时
- [ ] 完整流程时间合理（10-30秒）

## 有问题如何排查

1. 记忆不工作？
   → 检查是否显示了 📝记住了！
   → 检查 _keyFacts 字典里有没有内容

2. 工具总是不调用？
   → 检查用户问题是否真的需要工具
   → 看【分析中】阶段的输出

3. 还是超时？
   → 确认DeepSeek-R1模型是否正确加载
   → 看重试机制有没有触发 🔄第x次重试

4. 还有`<think>`？
   → 看最终答案里有没有
   → 确认用了新的SmartDialogSystem.cs
