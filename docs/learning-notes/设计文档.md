# 设计文档

## 一、设计目标

构建一个化工园区危化品合规审核AI Agent系统，实现：
- 化工合规知识的智能检索
- 多轮对话记忆管理
- 工业场景工具调用
- 符合等保三级安全要求

## 二、技术架构

### 2.1 核心组件

| 组件 | 说明 |
|------|------|
| RAGModule | 检索增强生成模块 |
| CoTModule | 思维链推理模块 |
| ReActModule | 推理-行动循环模块 |
| SessionService | 会话管理服务 |
| KnowledgeBaseService | 知识库服务（BM25检索） |
| HybridKnowledgeBaseService | 混合检索服务（BM25+向量） |

### 2.2 架构原则

- **依赖倒置**：高层模块依赖抽象接口
- **单一职责**：每个服务只负责一件事
- **开闭原则**：对扩展开放，对修改关闭

## 三、知识库设计

### 3.1 知识分类

| 类型 | 优先级 | 示例 |
|------|--------|------|
| 国标 | 高 | GB15603、GB30000 |
| 园区规则 | 中 | 存储管理规定、动火作业规范 |
| 历史案例 | 低 | 泄漏处置、安全检查整改 |

### 3.2 检索策略

1. **BM25粗召回**：基于关键词相关性
2. **向量检索补充**：基于语义相似性
3. **业务优先级重排序**：国标>园区规则>历史案例
