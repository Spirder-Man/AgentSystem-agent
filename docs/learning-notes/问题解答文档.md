# 问题解答文档

## Q1: 为什么需要RAG？

**A:** RAG（检索增强生成）解决了大模型无法访问私有知识的问题。在化工合规场景下：
- 法规条文需要精确检索
- 历史案例需要相似匹配
- 回答需要有据可依

## Q2: 为什么用BM25而不是纯向量检索？

**A:** 因为化工场景的特点是：
- 法规条文有明确的关键词（如"储罐"、"动火"）
- 需要精确匹配条款编号
- 向量检索适合语义相似，但关键词匹配同样重要

## Q3: 业务优先级如何实现的？

**A:** 在检索结果上叠加优先级分数：
```csharp
finalScore = bm25Score + priorityBonus
// 国标: +3000分
// 园区规则: +2000分
// 历史案例: +1000分
```

## Q4: 为什么需要会话记忆？

**A:** 多轮对话需要上下文：
- 用户问"刚才的温度是多少"需要记住之前的结果
- 对话过程中的关键事实需要持久化
- 工业场景的诊断建议需要基于历史数据
