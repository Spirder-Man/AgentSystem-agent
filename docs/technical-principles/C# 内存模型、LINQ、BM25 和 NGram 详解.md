# C# 内存模型、LINQ、BM25 和 NGram 详解

## 一、C# 内存模型：堆 vs 栈

### 1.1 核心结论

- **栈（Stack）**：存储值类型和引用类型的引用（指针），方法执行完自动释放，速度极快
- **托管堆（Managed Heap）**：存储所有引用类型的实际对象，由GC管理

### 1.2 断电后数据会丢吗？

是的！堆和栈都是"易失性内存"，断电后数据都会消失。

## 二、集合底层原理

### 2.1 倒排索引 vs 红黑树

- **倒排索引**：底层通常是Dictionary（哈希表），不是红黑树
- **红黑树**：是SortedDictionary的底层结构，用于需要按键排序的场景

### 2.2 哈希表的桶（Bucket）

桶是哈希表的基础存储单元：
1. 计算键的哈希值
2. 根据哈希值放到对应的桶
3. 冲突时用链表/红黑树挂接

## 三、LINQ用法

### 3.1 LINQ vs ORM

- **ORM（如EF Core）**：对象-关系映射框架，操作数据库
- **LINQ**：C#的查询语法，可操作内存集合、XML、数据库等

### 3.2 常用方法

- `Where`：过滤元素
- `Select`：投影/转换
- `OrderBy/OrderByDescending`：排序
- `GroupBy`：分组

## 四、NGram分词

### 4.1 中文分词挑战

中文无空格分隔，需要NGram分词

### 4.2 多粒度分词示例

```
"危化品储罐安全距离"
- 完整词：保留整句
- 2-gram："危化","化品","品储","储罐"...
- 单字："危","化","品"...
```
