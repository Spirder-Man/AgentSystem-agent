## 🔍 详细错误记录

### 错误1：向量扩展未安装

**错误ID**: VDB-001

**场景**: 首次尝试使用向量查询时

**现象**:
```
ERROR:  operator does not exist: vector <-> vector
LINE 1: SELECT embedding <-> '[0.1,0.2,0.3]'::vector FROM ...
```

**位置**: PostgreSQL 数据库层

**根本原因**:
- pgvector 扩展未在数据库中安装
- 未执行 `CREATE EXTENSION vector;`

**修复方案**:
```sql
-- 在目标数据库中执行
CREATE EXTENSION IF NOT EXISTS vector;

-- 创建向量表时确保扩展已安装
CREATE TABLE IF NOT EXISTS chemical_documents (
    id SERIAL PRIMARY KEY,
    content TEXT NOT NULL,
    embedding vector(768),
    regulation_type VARCHAR(100),
    priority INTEGER,
    source_file VARCHAR(255),
    chemical_type VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建HNSW索引优化检索性能
CREATE INDEX IF NOT EXISTS idx_chemical_embedding ON chemical_documents USING hnsw (embedding vector_cosine_ops);
```

**验证**:
```sql
-- 检查扩展是否安装成功
SELECT * FROM pg_extension WHERE extname = 'vector';

-- 测试向量运算
SELECT '[0.1, 0.2, 0.3]'::vector <-> '[0.4, 0.5, 0.6]'::vector;
```

### 错误2：Npgsql向量类型映射失败

**错误ID**: VDB-002

**场景**: 尝试将向量数据写入数据库时

**现象**:
```
Writing values of 'Pgvector.Vector' is not supported for parameters having no NpgsqlDbType or DataTypeName.
```

**位置**: `Services/DatabaseService.cs`, 第306行

**根本原因**:
- Npgsql 8.x 版本不支持旧版 `GlobalTypeMapper.UseVector()` 方法
- 向量参数没有正确映射到 PostgreSQL 的 vector 类型

**修复方案**:
```csharp
// 方案1：使用字符串格式转换（最终采用）
if (embedding != null)
{
    var vectorString = "[" + string.Join(",", embedding.Select(x => x.ToString("G", System.Globalization.CultureInfo.InvariantCulture))) + "]";
    command.Parameters.AddWithValue("@embedding", vectorString);
}

// SQL语句中显式转换
var sql = @"
    INSERT INTO chemical_documents (content, embedding, regulation_type, priority, source_file, chemical_type)
    VALUES (@content, @embedding::vector, @regulationType, @priority, @sourceFile, @chemicalType);
";
```

**关键代码位置**: `Services/DatabaseService.cs:303-317`

### 错误3：API端点方法错误

**错误ID**: VDB-003

**场景**: 调用 Ollama 嵌入API时

**现象**:
```
Response status code does not indicate success: 405 (Method Not Allowed).
```

**位置**: `Services/LlmService.cs`, 第255行

**根本原因**:
- 代码中使用了错误的 HTTP 方法（实际代码用POST，但日志显示GET）
- URL 拼接错误导致双斜杠 `http://localhost:11434//api/embeddings`

**修复方案**:
```csharp
// 使用Uri类正确拼接URL
var baseUrl = ModelConfig.Endpoint;
var url = new Uri(baseUrl, "/api/embeddings").ToString();

// 确保使用POST方法
var response = await _httpClient.PostAsync(url, content);
```

**关键代码位置**: `Services/LlmService.cs:240-256`

### 错误6：连接重复打开

**错误ID**: VDB-006

**场景**: 写入向量数据时

**现象**:
```
Connection already open
```

**位置**: `Services/DatabaseService.cs`, 第224行

**根本原因**:
- 在 `CreateConnection()` 方法中提前打开了连接
- 但 `GetConnectionAsync()` 又尝试打开一次

**修复方案**:
```csharp
private NpgsqlConnection CreateConnection()
{
    var connection = new NpgsqlConnection(BuildConnectionString());
    // 不在此打开连接，让调用方负责
    return connection;
}
```

**关键代码位置**: `Services/DatabaseService.cs:221-225`

### 错误7：向量维度不匹配

**错误ID**: VDB-007

**场景**: 插入向量数据时（潜在风险）

**现象**:
```
vector dimension mismatch
```

**位置**: 数据库插入操作

**根本原因**:
- 向量维度与表定义的 `vector(768)` 不匹配

**预防方案**:
```csharp
// 确保嵌入维度与配置一致
public int EmbeddingDimension { get; set; } = 768;

// 插入前验证维度
if (embedding != null && embedding.Length != _vectorConfig.EmbeddingDimension)
{
    throw new ArgumentException($"嵌入维度不匹配，期望 {_vectorConfig.EmbeddingDimension}，实际 {embedding.Length}");
}
```

## 📝 经验总结

### 成功经验

1. **向量类型处理**: 当 Npgsql 的 `Pgvector.Vector` 类型映射出现问题时，可以采用字符串格式转换作为备选方案
2. **API调试**: 添加详细的调试日志可以快速定位API调用问题
3. **模型管理**: 在使用嵌入模型前，务必确认模型已正确拉取

### 注意事项

1. **维度一致性**: 嵌入模型的输出维度必须与数据库表定义的维度一致
2. **索引创建**: 大量数据时必须创建HNSW索引，否则向量检索性能极差
3. **错误处理**: 向量检索失败时应有降级方案（如仅使用BM25检索）
