# 深度拆解：C#底层+检索算法核心原理（全维度答疑）

## 一、C# 内存基础：堆/栈都是内存

### 1.1 核心结论：堆/栈都是内存，只是存储规则不同

|区域|存储内容|访问速度|典型例子|
|---|---|---|---|
|栈（Stack）|① 值类型 ② 引用类型的引用 ③ 方法调用上下文|极快（纳秒级）|`int age=20;`|
|托管堆（Managed Heap）|① 所有引用类型（List/Dictionary/string）② 大的结构体|快（比栈稍慢）|`List<string> list=new List<string>();`|

**关键补充**：
- 堆和栈都是内存（RAM）的一部分
- 避免IO是指避免磁盘IO，堆/栈操作都是内存操作

## 二、C# 常用集合的底层原理

|集合类型|底层结构|核心特性|检索系统中的作用|时间复杂度|
|---|---|---|---|---|
|List<T>|动态连续数组|有序、可重复、索引访问快|存储文档列表、分词结果|访问O(1)，新增O(1)|
|Dictionary<TKey, TValue>|哈希表|键唯一、键值对映射、查找快|倒排索引、词频统计|增删改查O(1)|
|HashSet<T>|哈希表|元素唯一、无重复、查找快|去重|增删查O(1)|
|SortedList<TKey, TValue>|有序数组+哈希表|键有序+查找快|需排序的键值对场景|查找O(1)|

## 三、倒排索引设计

### _termDocFreq 结构

```csharp
Dictionary<string, Dictionary<int, int>>
// 外层key: 分词（token）
// 内层Dictionary<int, int>: key=文档ID, value=该词在文档中的出现次数
```

### 倒排索引查询流程

1. 用户查询 → Tokenize分词
2. 遍历每个分词token
3. 在倒排索引中查找该token对应的文档列表
4. 合并所有相关文档ID（去重）
5. 计算每篇文档的BM25得分

## 四、BM25算法原理

### 4.1 核心公式

```csharp
score = Σ [IDF(token) × TF饱和函数(token, doc)]
```

### 4.2 IDF（逆文档频率）

```csharp
idf = Math.Log((N - df + 0.5) / (df + 0.5) + 1)
// N: 总文档数
// df: 包含该词的文档数
```

**作用**：稀有词权重高，常见词权重低

### 4.3 TF饱和函数

```csharp
tfComp = (tf * (K1 + 1)) / (tf + K1)
// K1: 控制词频饱和度（默认1.2-1.5）
```

**作用**：词频增加带来的收益递减

### 4.4 文档长度归一化

```csharp
tfComp = (tf * (K1 + 1)) / (tf + K1 * (1 - B + B * (len / avgLen)))
// B: 归一化强度（默认0.75）
// len: 当前文档长度
// avgLen: 平均文档长度
```

## 五、业务优先级重排序

### 最终得分计算

```csharp
finalScore = bm25Score + priorityBonus
// 国标: +3000分
// 园区规则: +2000分
// 历史案例: +1000分
```

### 设计优势

1. 算法与业务分离：BM25负责相关性，业务层负责优先级
2. 保证合规优先级：国标文档优先返回
3. 粗召回+精排序：既保证召回率，又保证合规性
