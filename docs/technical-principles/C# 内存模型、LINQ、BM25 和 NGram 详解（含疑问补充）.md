# C# 内存模型、LINQ、BM25 和 NGram 详解（含疑问补充）

## 一、核心问题解答

### Q1：不管是存储在堆上还是栈上都是写入内存吗？

是的！堆和栈都是内存（RAM）的一部分，只是被CLR划分成了两个不同的区域，执行不同的存储职责。

### Q2：除了List、字典，还有哪些底层协调资源的集合/类型？

|集合|底层结构|特性|适用场景|
|---|---|---|---|
|ConcurrentDictionary|哈希表|线程安全|多线程场景|
|SortedSet|红黑树|有序+去重|需要排序的去重|
|Queue|队列|FIFO|任务排队|
|Stack|栈|LIFO|方法调用上下文|

## 二、HashSet去重原理

### 2.1 HashSet写的是什么？

`HashSet<int>`存储的是唯一的整数值（如文档ID），底层是一个"没有值的Dictionary"。

### 2.2 去重步骤

1. 初始化空HashSet
2. 遍历文档ID列表
3. 调用`Add(docId)`：
   - 计算哈希值
   - 定位到桶
   - 检查是否已有相同元素
   - 有则忽略，无则添加
4. 最终得到无重复的文档ID列表

## 三、BM25公式推导

### 3.1 IDF公式

```csharp
idf = Math.Log((N - df + 0.5) / (df + 0.5) + 1)
```

### 3.2 TF饱和函数

```csharp
tfComp = (tf * (K1 + 1)) / (tf + K1)
```

### 3.3 长度归一化
