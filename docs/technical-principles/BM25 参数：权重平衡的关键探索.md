# BM25 参数：权重平衡的关键探索

## 一、栈 vs 堆：理解引用类型存储

**引用类型 = 栈存地址，堆存数据**

- **栈** = 你口袋里的快递单（小、快、用完就扔）
- **堆** = 快递仓库（大、慢、存东西）

**List<string> list = new List<string>();**
- `list`：快递单（栈上），只写一个地址
- `new List<string>()`：仓库里的包裹（堆上），里面装所有字符串数据

## 二、为什么用对数？权重平衡

### 不用对数的问题

假设总文档 N=1000，稀有词 df=1：
- 原始公式：`(N-df+0.5)/(df+0.5) + 1 ≈ 667`
- 常见词 df=500：结果≈2

**问题：稀有词权重是常见词的300倍，会导致检索跑偏**

### 对数的作用

- `log(667) ≈ 7.6` → 稀有词权重
- `log(2) ≈ 0.7` → 常见词权重
- **权重差从300倍 → 10倍**，平衡不跑偏

## 三、BM25公式解析

### BM25核心公式

```csharp
score += idf * tfComp;
// 其中：
// idf = Math.Log((N - df + 0.5) / (df + 0.5) + 1)
// tfComp = (tf * (K1 + 1)) / (tf + K1 * (1 - B + B * (len / avgLen)))
```

### 参数作用

| 参数 | 默认值 | 作用 |
|------|--------|------|
| K1 | 1.2-1.5 | 控制词频饱和度 |
| B | 0.75 | 控制文档长度归一化程度 |

## 四、业务优先级重排序

### 为什么需要业务优先级？

BM25算法得分只反映文本相关性，但化工合规场景还需考虑：
- 国标 > 园区规则 > 历史案例
- 优先级：国标+3000分，园区规则+2000分，历史案例+1000分

### 最终得分计算

```
最终得分 = BM25得分 + 优先级加分
```
