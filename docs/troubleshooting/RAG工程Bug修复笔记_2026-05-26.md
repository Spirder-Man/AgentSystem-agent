## 修复详情

### Bug 1 — 重排序 key 对不上

**文件**: `ChemicalRAG.cs` — `SearchAsync`

```csharp
// ❌ priorityLevels key 是 "国标"/"园区规则"/"历史案例"（RegulationType 值）
// 但查找的是 Metadata["Priority"]，值是 "高"/"中"/"低"
r.Metadata["Priority"]   // "高" → 字典里没有 → priority 永远为 0

// ✅ 修复
r.Metadata["RegulationType"]   // "国标" → 字典里有
// 同时补上 "企业制度" → 500
```

## 审查经验总结

| 审查维度 | 问自己 |
|----------|--------|
| Key 一致性 | 存的 key 和查字典的 key 是同一组吗？ |
| 子系统完整性 | 类有几个存储？清理了几个？写入了几路？ |
| 降级副作用 | catch 返回的兜底值会进数据库/被二次使用吗？ |
| 计数器守恒 | 事件 1 次，计数器 +1 还是 +N？ |
| 路径对称性 | 不同格式/分支的步骤数一样吗？ |
| 量纲一致性 | 两个数要加在一起，数量级差多少倍？ |
| 参数穿透 | 工具调用链路上 userInput 每层都传到位了吗？ |
| 并发安全 | 读写同一集合时都加锁了吗？ |
| null 防御 | 所有 `.Substring` / `.Length` / `.ContainsKey` 前判空了吗？ |
